Git - Cartão de Referência

Help

git help [verbo]
git [verbo] --help
man git-[verbo]

Configuração Inicial

git config [opção] [parâmetro]
opção: --system (todo o sistema)
edita o arquivo: /etc/gitconfig

opção: --global (apenas o usuário)
edita o arquivo: ~/.gitconfig

opção: --list
Mostra configurações atuais do ambiente git

Edite o arquivo: [repositorio]/gitconfig
Configuração válida apenas para aquele repositório

[parâmetro]: alias.{nome_alias} {'comando'}
Cria um alias para um comando específico.
Exemplo:
git config --global alias.unstage 'reset HEAD --'

Repositórios

git init
Inicializa repositório no modo working dir (.git dir)

git init --bare
Inicializa repositório no modo repositório (compartilhável)

git clone [origem] [destino]
clona repositório e seus atributos (local ou remoto)

git remote [opções]
verifica e configura repositórios remotos ou no mesmo filesystem

opção: -v (apenas o usuário)
Mostra detalhes dos remotos

git remote add [nome] [url]
adiciona um repositório remoto ao projeto atual

Versionamento

git status
- verifica o estado do seu working dir.
(untracked, unmodified, modified, staged)
opção: -s ou --short - Mostra versão compacta do status

git rm [arquivo] [opções]
- remove arquivos do repositório e do seu working dir.

git commit [opções]
- grava arquivos do stage para o repositório

Sem parâmetros abre o editor de texto para que você escreva sua mensagem de commit

git log [opções]
- mostra histórico de commits

opção: --stat - mostra pequeno resumo de alterações nos arquivos opção: --pretty= - diversas opções de formatação do log - parâmetro: oneline – mostra uma lista abreviada de commit + mensagem de commit - parâmetro: format: “” - Formata saída do log conforme preferência do usuário. Exemplo para teste: 

            git log --pretty=format:“%h - %an, %ar : %s”

git tag - marca versão do software (release point), no commit atual opção: -a – cria uma tag anotada (mais informações são adicionadas)
